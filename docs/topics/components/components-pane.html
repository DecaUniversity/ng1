When using Components, camelCase used in the name of the component or in the name of its bindings translates into kebab-case in the markup and attributes.

.component("editableField", {bindings: {fieldTitle: "<"}}) -> <dh-editable-field field-title="">

Hero <hero> is a simple component. Its attribute receives a string.

This is mainHero.vMainController: {{mainHero}}

This block uses the component <dh-editable-field>. mainHero is passed through its field-value attribute. field-value has a one-way binding, <. The changes made to mainHero in the component do not reflect back to MainController.

This is hero1.vMainController: {{hero1}}

The list of heroes resides in the <dh-hero-list> controller.

Data is sent from parent to descendant component, one way, through the binding. Component modifies data and sends it back to parent.

Parent determines what to do with the modified data (further modify it, apply it to the model, etc).

For example: Use "Seattle" as the Location. The <dh-editable-field> component first records the change to "Seattle". However, it doesn' change the <dh-hero-detail> model. Through an event, it sends "Seattle" up to <dh-hero-detail>. <dh-hero-detail> controller has business logic that modifies the location to "Starbucks City". <dh-hero-detail> sends that up to the list component. The list component has no idea that the location was ever "Seattle". It receives "Starbucks City". <dh-hero-list> applies the change to its model - the highest structure to hold the list data. The upward cascading of data through events stops there.

Directives can require the controllers of other directives to enable cross-directive communication. This is achieved through the require property which creates an object mapping between a property key (local required controller name) and property value (name of component with the required controllers)

Note that the required controllers will not be available during the instantiation of the controller, but they are guaranteed to be available just before the $onInit method is executed!
